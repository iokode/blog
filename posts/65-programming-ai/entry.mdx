The content of this blog is written in [Markdown](https://commonmark.org/help/), but for some time now I have been thinking about switching to a format that is just as lightweight, but a bit more powerful*. Among other things, I am interested in it being an extensible format. My colleague [Alejandro Quirós](https://www.linkedin.com/in/alejandroquirospascual/) recommended that I take a look at [Asciidoc](https://asciidoc.org/), so I have spent some time looking into it and testing it locally.

Asciidoc seems to have everything I need. It is a lightweight text format that, by default, contains all the element types from Markdown and a few more, for example [*admonitions*](https://docs.asciidoctor.org/asciidoc/latest/blocks/admonitions/), which allow you to create warning or tip boxes.

Looking back at my posts, I have noticed that in some of them I comment on different tweets, so I thought it would be useful to be able to embed a tweet inside a post. Asciidoc does not allow you to embed a tweet by default, but because it is extensible, you can create a plugin for it.

The software that transforms Asciidoc documents into HTML documents is called [Asciidoctor](http://asciidoctor.org/) and it is written in [Ruby](https://www.ruby-lang.org/es/). Asciidoctor extensions are therefore written in Ruby.
